Introduction

Open Data Kit (ODK) is a group of software tools for managing mobile data collection. A key component of ODK is offline data collection–the ability to collect data without internet connectivity.

ODK System Components

Questionnaires

Questionnaires start as paper documents or word files. These are conceptual models of what questions need to be asked, and in what order. Once the forms are completed, someone with ODK questionnaire experience converts the questions and expected responses into a Microsoft Excel document using the XLSForm design. This is a precise format for converting questionnaires into ODK electronic format using Microsoft Excel.

Server

Storage and Management

  1. Server Overview

    The Remote Server, also referred to as a Cloud Server, is a physical machine or virtual machine that can be purchased, initiated, and configured through a web browser. The advantages are high-availability, low cost, and consistency. Our server model uses a remote server from the company DigitalOcean.com.

  2. Server Components

    Below are descriptions of main components of the Remote Server: Operating System (OS), Database, Web Server.

    1. OS: Linux Ubuntu. An OS is the underlying platform software which controls the server. Choosing Linux means a choice of the family of software that can run on the Linux platform. The software in the Linux family is open source. There are no licensing fees with this software. Apple and Microsoft software will not run on a Linux OS.
    2. Web Server: Tomcat. The Web Server manages all the web sites on a server. In our system, the main website hosted on the Remote Server is the ODK Aggregate site.
      • Forms can be uploaded as Xform files.
      • Tablets download the blank forms, collect data, then upload.
      • Data can be viewed and downloaded.
      • User account management

    Additional optional websites include the Dashboard and the Admin interface.

  3. Database: MySQL (by Oracle). This is the main data repository on the server. ODK Aggregate uses a database in MySQL to store form definitions, user Accounts, and form data. MySQL is a database management system. It is not limited to storing ODK data. It can store data, tables, or databases from other software or custom web applications.

Tablets

Electronic tablets or Smartphones with the Android operating system are required to collect data. Internet connectivity is not necessary for data collection, but it is required for software installation, configuration, form download, and data submission.

ODK Collect must be downloaded from the Google Play store and installed on the device.

Once ODK Collect is installed, ODK Collect is configured with the internet address (HTTP address) of the server and with a user name and password. The user name and password must be first created on the Server ODK Aggregate website.

Once the server connection is established, forms are downloaded to the tablet. The table can disconnect from the internet and data collection can proceed.

At the completion of data collection, or at regular intervals, the completed forms must be uploaded to the server. The tablet must be connected to the internet, the completed forms are submitted, and the data is removed from the tablets.
